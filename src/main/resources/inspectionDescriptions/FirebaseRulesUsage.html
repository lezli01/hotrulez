Reports suspicious usage in a Firebase Cloud Firestore Security Rules file: an allow rule with no if condition (which unconditionally grants the operation), a recursive wildcard {name=**} used in a match path without rules_version = '2'; (its match semantics differ between rules versions), and a single-path helper (get, getAfter, exists, existsAfter) called with an argument count other than one.

These checks describe the syntax and structure of the rules file only. They do not evaluate whether a rule is secure or whether it authorizes any particular request.
