Reports unresolved and unused symbols in a Firebase Security Rules file, using the same scope-based resolution as go-to-definition and rename. An identifier that does not resolve to any function, parameter, let binding, or path/wildcard variable in scope — and is not a built-in such as request, resource, or a path helper — is reported as Cannot resolve symbol; an unresolved call additionally offers to create the missing function. A function or let binding that nothing references is reported as unused (with a quick-fix to remove it), and an unused function parameter is reported without a fix.

Member accesses (request.auth), path/wildcard variables, and built-ins are never reported, and the check stays quiet inside code the parser could not yet parse.

This check describes the symbols in the rules file only. It does not evaluate whether a rule is secure or whether it authorizes any particular request.
